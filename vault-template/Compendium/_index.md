# Compendium

> Active projects and ongoing responsibilities. Everything here is
> bounded: it has a goal, a scope, and an eventual end. The Custodian
> creates and closes project folders. Content agents create notes
> within them. When a project completes, the Custodian archives it
> to the Reliquary.

## Notes

- Each project lives in `Compendium/<project-slug>/`
- Every project folder has a `_brief.md` — the project's definition,
  goal, status, and deadline
- The Custodian creates the folder and brief shell; you fill in the
  brief
- Notes within a project folder are owned by the project — they do
  not need `domain` frontmatter
- When a project completes, notify the Custodian to archive it to
  `Reliquary/<YYYY>/<project-slug>/`
- A note that would remain meaningful after the project ends should
  be moved to `Codex/` — the Quaestor handles this during archiving
